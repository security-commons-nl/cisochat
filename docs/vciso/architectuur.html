De vCISO-dirigent

Hoe één AI-brein de volle breedte van het CISO-werk nabootst: redeneren langs NIST CSF 2.0, gegrond in de normen, en open-source tooling aansturen — adviserend, de mens beslist.

NIST CSF 2.0 open source · EU-soeverein dirigent, geen doos status: blueprint vastgesteld — nog niet gebouwd

Hoe een vraag door de dirigent stroomt

Van vraag tot onderbouwd advies. De dirigent classificeert, kiest een instrument, en passeert vóór elke onomkeerbare stap een harde, code-niveau gate — niet het model dat "nee" zegt.

Input

Gebruiker stelt een vraag

"Moeten we een DPIA doen?" · "Hoe weeg ik deze BIO-maatregel?" · "Zijn we compliant?" — vragen die geen Google-zoekopdracht beantwoordt.

Stap 1 · routing

CSF-functie-detectie

LLM-classificatie: valt dit onder Govern · Identify · Protect · Detect · Respond · Recover — of is het puur advies?

Stap 2 · routing

Capability-routing

Welke vCISO-capability is aan de orde, en welk invullingstype hoort erbij?

Stap 3 · instrumentkeuze
[Brein]

Redeneren + RAG

Geen tool — de dirigent zelf, gegrond in de kennisbank (normen).

[OSS]

Instrument aanroepen

Extern open-source tool via MCP of REST/CLI-adapter.

[Bouwen]

Eerlijk melden

Geen volwassen OSS — benoem de gap i.p.v. verzinnen.

Stap 4 · Noorderster

⬢ Deterministische routing-gate

Vóór elke schrijf- of onomkeerbare actie: een harde check op code-niveau. AI bereidt voor, de mens beslist. (LLM-guardrails falen onder aanvallen — daarom geen "vraag het netjes aan het model".)

Stap 5 · output

Onderbouwd advies  +  auditspoor

Antwoord mét bronverwijzing en afweging. Elke prompt, tool-aanroep, bron en beslissing landt in een herleidbaar, append-only spoor → terug naar de gebruiker.

Gelaagde architectuur

Van interface tot instrumenten. Het brein redeneert bovenop een grounding-laag; een router kiest het instrument; een deterministische gate bewaakt de Noorderster; de instrumenten zijn bestaande open-source tools.

UI Interface

Chat als ingang; op termijn ingebed in of gekoppeld aan een GRC-frontend.

01 Dirigent-brein

CISO-persona · NIST CSF-redeneerframe · advies met onderbouwing, transparant over onzekerheid — beslist nooit.

02 Capability-router

CSF-detectie → capability → instrumentkeuze ([Brein] / [OSS] / [Bouwen]). Progressive disclosure van de skill-laag.

03 Grounding · RAG

Kennisbank met de normen: BIO 2.0 · ISO 27001/27701/22301 · AVG · EU AI Act. Antwoorden steunen op opgehaalde bronnen, niet op trainingsdata.

04 Deterministische gate + auditspoor

Harde code-niveau check vóór onomkeerbare acties (Noorderster) · tamper-evident, append-only beslislog.

05 Instrumenten

Externe open-source tooling per CSF-functie, aangestuurd via MCP of REST/CLI — plus [Brein] waar geen volwassen tool bestaat.
